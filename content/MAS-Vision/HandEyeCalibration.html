手眼标定

概述

手眼标定用于建立相机坐标系与机器人坐标系之间的几何关系，其目标是：

将视觉系统得到的“目标在相机坐标系下的位置"，转换为机器人能够直接使用的“目标在机器人基坐标系下的位置

  • 建立：相机坐标系 ↔ 机器人坐标系的刚体关系
  • 求解：相机与机器人之间的外参

公式

  • 手眼标定（眼在手上）：
    • T_base_to_target = T_base_to_tool × T_tool_to_cam × T_cam_to_target
  • 手眼标定（眼在手外）：
    • T_base_to_target = T_base_to_cam × T_cam_to_target

类型

类型 相机是否随末端动 手眼标定结果
眼在手上（Eye-in-Hand） 是 T_tool_to_cam
眼在手外（Eye-to-Hand） 否 T_base_to_cam

变换符号

  • T_base_to_target: 机器人基坐标系 → 目标坐标系
    • 含义：目标在机器人基坐标系（Base）下的位姿
    • 用途：最终下发给机器人 / PLC 使用
  • T_base_to_tool: 机器人基坐标系 → 机器人末端坐标系
    • 含义：机器人末端（Tool）在机器人基坐标系下的当前位姿
    • 来源：机器人控制器 / PLC 实时读取
    • 特点：运行时实时变化
  • T_tool_to_cam: 机器人末端坐标系 → 相机坐标系
    • 含义：相机相对于机器人末端的固定安装关系
    • 来源：眼在手上手眼标定结果
    • 特点：固定不变
  • T_base_to_cam: 机器人基坐标系 → 相机坐标系
    • 含义：相机在机器人基坐标系下的位姿
    • 来源：眼在手外手眼标定结果
    • 特点：固定不变
  • T_cam_to_target: 相机坐标系 → 目标坐标系
    • 含义：目标相对于相机坐标系的位姿
    • 来源：视觉算法输出
    • 常见形式
      • 2D：(X, Y, θ)
      • 3D：(X, Y, Z, Rx, Ry, Rz)
      • 说明：表示相对相机的偏移量，不是绝对世界坐标

职责划分

  • 视觉系统
    • 只负责输出T_cam_to_target
    • 不关心机器人、基坐标、手眼关系
  • DM（中间调度/算法模块）
    • 读取机器人当前位姿T_base_to_tool
    • 持有手眼标定结果
    • 计算T_base_to_target
    • 可进一步叠加抓取偏置
  • 机器人/PCL
    • 只接收并执行T_base_to_target或抓取位姿

抓取补充

手眼标定只能解决“看准”，不能保证“抓准”

  • 在实际抓取中，通常需要引入抓取偏置：
    • T_base_to_grasp = T_base_to_target × T_target_to_grasp
  • T_target_to_grasp：
    • 含义：抓取点（夹爪 TCP）相对于目标基准的固定偏移
    • 来源：
      • CAD / 图纸（初值）
      • 自动校正治具
      • 少量人工示教

标定算法

Eye-in-Hand（眼在手上）

数学模型：

  • A X = X B
    • A：机器人末端相对运动
    • B：标定板在相机坐标系下的相对运动
    • X：末端到相机的固定安装外参

算法列表

  • 两步法（Tsai-Lenz）：先解旋转再解平移，快且稳，默认首选
  • 对偶四元数（Park / Daniilidis）：用四元数/对偶四元数整体建模旋转（及平移），精度高但更吃数据质量与姿态覆盖
  • 正交矩阵分解（Horaud）：用 SVD 等方法在正交约束下求旋转，旋转更“干净”，适合高精度
  • 改进型解耦（Andreff）：更联合地处理旋转和平移，样本较少时往往更稳
  • 所有算法的差别，本质是：
    • 旋转/平移的数学表示方式
    • 误差最小化策略
    • 分步求解或联合求解

常见问题

Warning

误解

  • 标手眼标定 ≠ 夹爪 TCP 标定
  • 手眼标定 ≠ 抓取策略
  • 视觉定位精度 ≠ 最终抓取精度

手眼标定仅解决坐标映射问题，实际抓取效果仍依赖机械结构、TCP 定义与抓取偏置建模

Warning

机器人位姿参与不充分

当用于手眼标定的机器人位姿分布过于集中时，求解的刚体变换将出现退化

常见表现包括：

  • 标定在当前拍照点附近看似准确
  • 机器人换位后误差明显放大

手眼标定应覆盖足够多样的末端姿态（平移 + 旋转），避免仅在局部空间内拟合

Tip

区分“手眼误差”与“抓取误差”

手眼标定只建立相机与机器人之间的几何关系，并不包含夹爪 TCP 定义或抓取点偏置

当误差在不同目标位置下保持一致时，应优先检查：

  • TCP 是否准确
  • 是否缺失抓取偏置T_target_to_grasp

避免将抓取问题错误归因于手眼标定失败

Tip

标定完成后缺乏最小可用性验证”

手眼标定完成后，建议至少进行以下验证：

  • 在不同机器人位姿下重复计算同一目标的T_base_to_target
  • 验证结果在 Base 坐标系下的一致性

通过验证的标定结果，才应被用于正式运行

Tip

眼在手上与眼在手外的选择建议

  • 眼在手上（Eye-in-Hand）适用于：
    • 多拍照位
    • 跟随抓取
    • 工位变化频繁的场景
  • 眼在手外（Eye-to-Hand）适用于：
    • 工位固定
    • 结构简单
    • 调试周期短
