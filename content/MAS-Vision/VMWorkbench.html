MAS.Vision - VM Workbench

兼容

目前存在以下问题：

  1. VM 的全局样式影响 DM 部分控件样式，如按钮
  2. DM 的全局样式影响 VM 部分控件样式，如输入框
  3. VM 脚本编辑器内部依赖了某些老 API，而这些 API 在 .NET 8 上不支持
  4. 双击脚本模块时引发的错误：System.Windows.Markup.XamlParseException: The invocation of the constructor on type 'ShellModuleControl.ShellCodeEditor' that matches the specified binding constraints threw an exception.

集成

由于 VM SDK 部分组件依赖 .NET Framework，当前无法在 .NET 8 环境中完全兼容运行

因此采用“独立进程”方式集成：

  • VisionMaster 作为独立应用运行（.NET Framework）
  • DM（.NET 8）通过环境变量检测并启动 VisionMaster
  • 两者通过进程级解耦，避免运行时冲突
  • 环境变量仅表示“已注册”，有时需要配合File.Exists校验
