WebView 事件与审批

MAS.PrismaTalk 的对话 UI 由 WPF 后端通过 WebView 执行 JavaScript 驱动。普通文本回复、Agent timeline、工具审批都在同一个 WebView 对话流中呈现

后端调用前端

PrismaTalkClient 继承自 WebView 服务基类，通过 ExecuteScriptAsync 调用前端函数

C# 方法 前端函数 说明
DisplayUserMessageAsync addUserMessage(content) 添加用户消息
StartAssistantStreamAsync startAssistantStream(streamId) 创建助手流式消息容器
UpdateAssistantStreamAsync updateAssistantStream(streamId, chunk) 追加助手自然语言内容
EndAssistantStreamAsync endAssistantStream(streamId) 结束流式消息
ClearConversationAsync clearAllMessages() 清空对话
ShowToastMessageAsync showToastMessage(content, level) 显示 toast
SetThemeAsync setTheme(theme) 切换 WebView 主题

Agent Runtime 额外使用以下前端函数：

后端调用 前端函数 说明
EmitThinkingAsync showPlainThinking(streamId, message) 普通轻量思考状态，不创建 timeline
EmitAsync renderAgentEvent(streamId, evt) 渲染 Agent timeline 事件
WebViewToolApprovalService renderAgentApprovalRequest(payload) 渲染工具审批卡片

Agent timeline 事件

后端当前下发的事件主要有：

{
  "kind": "tool_call",
  "id": "call_0",
  "toolName": "cli.run",
  "provider": "cli",
  "risk": "high",
  "argsPreview": "{\"command\":\"dotnet build\"}",
  "status": "awaiting_approval",
  "elapsedMs": 123
}

{
  "kind": "tool_call",
  "id": "call_0",
  "status": "success",
  "summary": "命令执行成功",
  "elapsedMs": 2345
}

{
  "kind": "complete",
  "toolCount": 1,
  "elapsedMs": 3000
}

前端 renderAgentEvent 会懒创建 timeline。普通问答只显示 showPlainThinking 和最终流式文本；只有真实工具调用、资源读取、审批或观察结果才出现可折叠的 Agent 过程

审批卡片

高风险工具执行前，AgentLoop 会调用 IToolApprovalService。默认实现 WebViewToolApprovalService 会渲染如下 payload：

{
  "streamId": "...",
  "id": "call_0",
  "approvalId": "...",
  "toolName": "entity.delete",
  "provider": "database",
  "risk": "high",
  "argsPreview": "{\"instanceId\":\"...\",\"sql\":\"DELETE ...\"}",
  "title": "需要确认",
  "message": "该工具属于高风险操作。请确认参数无误后再允许执行。"
}

前端用户点击允许或拒绝后，通过 WebView postMessage 回传：

{
  "type": "toolApprovalResponse",
  "approvalId": "...",
  "approved": true
}

后端用 approvalId 匹配等待中的审批请求。拒绝时不会执行工具，而是把取消结果作为 ToolResult 回填给模型

前端职责边界

前端只负责展示和回传用户审批选择，不决定工具是否可执行。风险等级、参数校验、工具路由和执行都在后端完成。

这使得安全逻辑集中在 Agent Runtime 和各 ToolProvider 内，WebView 只承担可视化与交互职责。
