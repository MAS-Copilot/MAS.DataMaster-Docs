MAS.PrismaTalk

MAS.PrismaTalk 是 MAS.DataMaster 中的本地 AI 对话与 Agent 编排模块。当前实现已经从早期的“模型生成 C# 脚本并执行”演进为统一的 Agent Runtime：模型可以读取内置 Skill 文档、发现多来源工具、结构化调用工具、接收执行结果并继续推理，最后把自然语言结果返回给用户

当前定位

PrismaTalk 负责对话、工具目录汇聚、工具调用编排、WebView 展示和配置读取。真正依赖宿主上下文的能力，例如数据库实例、运行时脚本编译、设备访问等，优先由 MAS.DataMaster 宿主通过 NamedPipe IPC 暴露为内部工具

MCP 已经作为外部扩展入口接入，但不是内部能力的主干。内部 MAS 能力优先走 RemoteToolProvider + MAS.NamedPipeIpc，MCP 主要用于第三方工具、客户自定义独立工具进程、跨 Agent 生态复用等场景

关键源码入口

入口 说明
Agent/Core/AgentLoop.cs Agent 主循环，负责工具目录注入、模型调用、工具执行、审批和最终回答
Agent/Core/ToolRegistry.cs 聚合所有 IToolProvider，按工具名路由调用
Agent/Protocols/NativeToolCallProtocol.cs OpenAI 兼容 tools / tool_calls 结构化协议适配
Agent/Protocols/TextEnvelopeProtocol.cs 文本信封 fallback 协议
Agent/Providers/*ToolProvider.cs 各类工具来源
Services/PrismaTalkClient.cs OpenAI 兼容流式请求、tool_calls 增量解析和 WebView 脚本调用

与 MAS.DataMaster 的关系

PrismaTalk 是模块侧 Agent 编排器，MAS.DataMaster 是宿主侧业务能力执行环境。两者通过 MAS.NamedPipeIpc 通信：

  • PrismaTalk 通过 RemoteToolProvider 调用宿主的 tool.list 和 tool.invoke
  • MAS.DataMaster 在 NamedPipeIpc/IpcBootstrapper.cs 注册 skill.execute、tool.list、tool.invoke
  • 宿主侧 DatabaseToolProvider 以 entity.* 工具暴露数据库实例、表结构、查询和写操作
  • 旧的 C# 脚本链路仍保留为 script.csharp，用于兼容 Modbus、Siemens、Mitsubishi 等 Skill 脚本

设计原则

  • 内部能力优先使用强类型 ToolProvider 和 IPC，不把内部软件接口为了形式统一而强行 MCP 化
  • 高风险工具必须经过统一审批门，审批点在执行前，由 WebView 工具审批卡片完成
  • 普通问答保持轻量，不创建 timeline；只有真实工具调用、资源读取、审批等可审计动作才显示 Agent 过程
  • MCP 保留为高级外部扩展能力，默认无 server 配置时不影响普通对话和内置工具
