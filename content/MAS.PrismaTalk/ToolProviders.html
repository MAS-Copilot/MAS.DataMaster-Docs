工具系统

工具系统把不同来源的能力统一成模型可发现、可校验、可审批和可回填的 function calling 工具。

工具契约

每个工具定义包含：

字段 含义
Name Runtime 内唯一的真实工具名
Description 提供给模型的用途和调用约束
Parameters JSON Schema 参数定义
Risk Low、Medium 或 High
ProviderId 工具来源标识
IsHidden 是否需要先通过工具发现流程激活

调用请求包含真实工具名、JSON 参数和可选 provider；执行结果统一为 ToolResult：

{
  "isSuccess": true,
  "summary": "简短结果摘要",
  "data": {
    "id": "asset-1"
  },
  "error": null
}

data 在 provider 内可以是 JSON 文本或普通文本。回填模型时，合法的 JSON 对象和数组会恢复为结构化节点，避免二次转义；其他内容保持字符串。

注册与路由

工具注册表在首次使用时向所有 provider 请求目录，并按工具名建立路由。模型调用工具时，执行管线先解析 wire-safe 名称，再由注册表路由到目标 provider。

工具名不区分大小写。不同 provider 不应注册同名工具；外部工具应使用明确命名空间，避免覆盖 DM 内置能力。

两层工具发现

为了避免每轮把大量业务工具 schema 发送给模型，Runtime 使用“目录 + 活跃集”机制：

  • 基础工具和发现元工具直接进入活跃集；
  • 隐藏工具只在目录摘要中出现；
  • 模型通过 tool.search 检索相关工具；
  • 通过 tool.activate 把选中的工具加入本轮活跃集；
  • 激活结果写回会话，后续轮次自动恢复。

如果模型跳过激活而直接使用真实工具名，执行管线会在能够唯一解析时自动激活；无法解析时返回引导性错误，让模型重新搜索。

内置 provider

Provider 主要能力 定位
runtime 当前时间、应用信息、工作区信息 低风险环境上下文
skill 读取 Skill 说明和关联资源 按需加载领域知识
draft 草稿的创建、读取、编辑、删除、清空和列表 大内容分段生成与修改
plan 向用户展示结构化决策步骤并等待选择 复杂需求澄清，不是多 Agent 计划器
script 执行 C# 或 Python 脚本 兼容和计算能力，高风险
cli 命令执行与专用文件操作 开发诊断兜底能力
host DM 宿主通过 IPC 暴露的 typed tools 内部业务能力主干
mcp stdio MCP server 工具 第三方和客户扩展入口

DM 宿主工具

PrismaTalk 通过 IPC 获取宿主实时工具目录，而不是在对话进程中静态复制全部 DM 业务能力。宿主侧可以暴露数据库、QueryAsset、ChartAsset、ViewAsset 和其他依赖当前项目或运行环境的 typed tools。

IPC 交互遵循三个基础方法：

IPC 使用长度前缀 JSON 消息。外层请求包含关联标识、方法名和序列化后的负载字符串：

{
  "RequestId": "7f6f0f9b...",
  "Method": "ipc.ping",
  "PayloadJson": "{}"
}

{
  "RequestId": "a81d24e1...",
  "Method": "tool.list",
  "PayloadJson": "{\"ProviderId\":null}"
}

{
  "RequestId": "bd32c7a0...",
  "Method": "tool.invoke",
  "PayloadJson": "{\"ToolName\":\"entity.query\",\"ArgumentsJson\":\"{\\\"instanceId\\\":\\\"main\\\",\\\"sql\\\":\\\"SELECT * FROM Device\\\"}\",\"Provider\":null}"
}

宿主返回的工具定义直接进入统一目录，调用结果也转换为统一 ToolResult。因此宿主工具与本地工具共享相同的发现、审批、时间线和模型回填流程。

数据库 typed tools

数据库工具按业务动作拆分，典型能力包括：

工具 风险 说明
entity.listInstances Low 列出可用数据库实例
entity.listTables Low 列出实例中的表
entity.describeTable Low 读取表字段结构
entity.query Low 执行受限的只读查询
entity.insert Medium 插入数据
entity.update High 更新数据，通常要求条件
entity.delete High 删除数据，通常要求条件
entity.alterTable High 执行受约束的表结构变更

这些工具比让模型生成任意脚本更可控：参数 schema、风险等级、SQL 范围和业务校验都由宿主决定。

Skill 资源

Skill 是可读取的能力包，包含领域知识、格式约定、工作流和示例。Skill 目录只把名称和描述放进本轮前置上下文；模型需要详细内容时调用：

{
  "name": "skill.read",
  "arguments": {
    "name": "QueryAsset",
    "path": "SKILL.md"
  }
}

资源路径只能指向该 Skill 包已索引的说明、参考、示例、资产或模板。读取 Skill 不代表必须执行代码；产生业务副作用仍应优先调用 typed tool。

草稿工具

草稿用于承载不适合一次性放进工具参数的大文本。支持创建/覆盖、读取、精确编辑、删除、清空和列举。草稿属于中间工作区，不等同于已经写入 DM 项目或文件系统。

典型流程：

生成较长内容 → draft.put → 多次 draft.edit → draft.get 校验 → 交给 typed tool 正式应用

计划交互工具

计划工具用于一次性提交多个决策步骤，每个步骤包含一个问题和有限候选项。前端逐步收集用户选择，最终把结构化决策结果回填给模型。

它适合在实施前澄清数据库、资产类型、覆盖策略等关键选项。它不创建子 Agent，也不改变单 Agent Runtime。

CLI 与文件工具

CLI provider 包括通用非交互式 PowerShell 执行，以及读取、写入、复制和精确编辑文件的专用工具。

  • cli.run、写入、复制和编辑属于高风险操作；
  • cli.read_file 为中风险，只负责读取；
  • 命令具有超时限制；
  • stdout 和 stderr 会截断，避免过长结果污染上下文；
  • 专用文件工具优先于把内容和路径拼接成 shell 命令；
  • CLI 适合诊断和兜底，不应成为稳定 DM 业务集成的主接口。

脚本工具

脚本 provider 支持 C# 和 Python。脚本执行属于高风险能力，适合计算、数据分析或兼容现有自动化 Skill。能够通过宿主 typed tool 完成的业务操作，不应优先生成脚本。

MCP 工具

MCP provider 启动配置的 stdio server，完成初始化和 tools/list 后，将工具暴露为：

mcp.<server>.<tool>

MCP 工具默认使用高风险等级。只有明确确认只读且安全的工具才适合按名称降级。MCP 的配置、协议和风险建议见 MCP 集成。

风险边界

风险等级决定 Runtime 的统一执行政策，但 provider 仍必须执行参数和业务校验：

  • Low：直接调用；
  • Medium：直接调用，但工具应严格限制读取或变更范围；
  • High：执行前必须获得用户审批。

审批只代表用户允许尝试执行，不代表参数合法。路径范围、SQL 限制、资产状态、设备连接和业务权限仍由具体工具验证。
