工具提供者

IToolProvider 是 PrismaTalk Agent Runtime 的工具来源抽象。每个 provider 负责列出工具定义，并在被路由时执行对应工具

public interface IToolProvider {
    string ProviderId { get; }
    Task<IReadOnlyList<ToolDefinition>> ListToolsAsync(CancellationToken ct = default);
    Task<ToolResult> InvokeAsync(ToolCallRequest request, CancellationToken ct = default);
}

ToolRegistry 聚合所有 provider，并按工具名建立路由表。若 ToolCallRequest.Provider 为空，则按 ToolName 查找；若指定 provider，则优先按 provider id 解析

数据库 typed tools

宿主侧 DatabaseToolProvider 暴露以下工具：

工具 风险 说明
entity.listInstances Low 列出可用数据库实例 key
entity.listTables Low 列出实例中的表
entity.describeTable Low 读取表字段结构
entity.query Low 执行单条只读 SELECT/WITH，并强制 LIMIT
entity.insert Medium 执行单条 INSERT
entity.update High 执行单条 UPDATE，默认要求 WHERE
entity.delete High 执行单条 DELETE，默认要求 WHERE
entity.alterTable High 执行表级 DDL，拒绝数据库级操作

这些工具比让模型生成 C# 或任意 SQL 更可控：工具描述、参数 schema、风险等级和校验逻辑都由代码定义

CLI 的定位

cli.run 是高风险兜底工具，用于执行非交互式 PowerShell 命令。它适合开发、诊断、构建、查看文件等临时任务，但不应替代 typed tools

使用 CLI 时应注意：

  • 所有调用都会进入高风险审批。
  • 默认超时为 60 秒，最大 600 秒。
  • stdout / stderr 会截断，避免过长结果污染上下文。
  • 适合一次性诊断，不适合作为长期业务集成接口。

MCP 的定位

MCP 不是内部工具总线的替代品。对于内部 MAS 能力，优先使用 RemoteToolProvider、typed tools 或脚本兼容层。MCP 更适合：

  • 第三方工具生态
  • 客户现场自定义工具进程
  • 非 .NET 或不希望引用 MAS 主程序的独立服务
  • 希望同一工具同时被 PrismaTalk、其他桌面 Agent 或 IDE Agent 复用的场景
