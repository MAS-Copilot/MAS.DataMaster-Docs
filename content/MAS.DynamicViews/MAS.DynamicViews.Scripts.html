全局脚本

全局脚本是 MAS.DynamicViews 提供的工程级运行时扩展能力。

它允许动态视图工程在运行过程中执行统一的 C# 脚本逻辑，用于处理工程级公共行为，例如变量监听、导航监听、运行状态处理、公共业务逻辑等。

当前脚本系统只支持 全局脚本，不区分页面脚本和窗口脚本。

设计目标

全局脚本系统主要用于：

  • 提供工程级公共逻辑入口
  • 支持访问动态视图运行时服务
  • 支持监听变量变化
  • 支持监听页面导航变化
  • 支持执行变量读取和写入
  • 支持脚本内容变更后重新编译
  • 支持脚本内容未变化时复用已编译实例
  • 避免重复编译和重复初始化

脚本定位

全局脚本作用于整个动态视图工程。

动态视图工程
└── 全局脚本
    ├── 变量变化处理
    ├── 页面导航处理
    ├── 公共运行逻辑
    └── 公共辅助方法

全局脚本不是页面或窗口模型的一部分，而是工程运行时的公共扩展逻辑

脚本服务集合

脚本运行时通过DynamicViewScriptServices暴露可访问的服务

/// <summary>
/// 动态视图脚本服务集合
/// </summary>
/// <remarks>
/// 向全局脚本暴露便捷的且允许运行时访问的公开服务。
/// </remarks>
public sealed class DynamicViewScriptServices {
    /// <summary>
    /// 获取变量值提供者
    /// </summary>
    public required IDynamicViewVariableValueProvider VariableProvider { get; init; }

    /// <summary>
    /// 获取变量写入服务
    /// </summary>
    public required IDynamicViewVariableWriteService VariableWriter { get; init; }

    /// <summary>
    /// 获取动态视图导航服务
    /// </summary>
    public required IDynamicViewNavigationService NavigationService { get; init; }
}

脚本通过服务集合访问运行时能力，不直接依赖具体运行时实现

这样可以降低脚本与内部实现的耦合，也方便后续控制脚本可访问的能力范围

编译与缓存

全局脚本内容本质上是字符串。

为了避免每次运行时刷新都重复编译，运行时应根据脚本文本计算签名

读取全局脚本文本
  ↓
计算脚本哈希
  ↓
判断哈希是否变化
  ↓
未变化：复用已编译脚本
  ↓
已变化：重新编译并初始化

这样可以避免：

  • 重复编译造成性能浪费
  • 重复初始化导致事件重复订阅
  • 脚本未变化时产生不必要对象
  • 频繁刷新时出现卡顿

资源释放

全局脚本可能会订阅变量事件、导航事件或持有运行时服务引用。

因此脚本重新编译、工程关闭或运行时释放时，必须释放旧脚本资源。

释放时应处理：

  • 取消变量事件订阅
  • 取消导航事件订阅
  • 清空脚本服务引用
  • 释放旧脚本实例
  • 避免事件重复订阅

异常处理

全局脚本异常不应直接中断动态视图主流程

执行脚本逻辑
  ↓
捕获异常
  ↓
记录日志
  ↓
跳过本次脚本处理
  ↓
不中断页面显示、变量刷新、通讯流程

注意事项

使用全局脚本时需要注意：

  • 不要在脚本中执行长时间阻塞操作
  • 不要重复订阅事件
  • 重新编译前必须释放旧脚本实例
  • 变量写入应避免形成循环触发
  • 脚本异常应记录日志，不应影响主流程
  • 脚本服务只暴露必要能力，避免脚本直接操作内部对象
