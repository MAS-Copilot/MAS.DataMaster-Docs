MAS.DynamicViews.Variable

变量系统是 MAS.DynamicViews 的数据核心

整个动态视图运行过程中：

  • 控件绑定
  • 通讯同步
  • 运行时刷新
  • 状态显示
  • 事件交互

最终都依赖变量系统完成数据驱动

变量系统目标

变量系统主要目标：

  • 统一管理变量定义
  • 支持结构体变量
  • 支持字段级绑定
  • 支持变量监控
  • 支持批量更新
  • 支持通讯映射
  • 支持运行时绑定
  • 支持引用追踪
  • 支持高频实时刷新

变量模型

当前变量系统采用结构体变量 + 字段模型架构

整体结构：

DynamicViewVariableModel
├── VariableId
├── Name
├── GroupName
├── AccessMode
├── IsEnabled
├── Description
├── Fields
└── Properties

其中：

属性 说明
VariableId 结构体变量唯一标识
Name 结构体名称
GroupName 结构体分组
AccessMode 结构体访问模式
IsEnabled 是否启用
Description 结构体描述
Fields 字段定义集合
Properties 扩展属性

为什么使用结构体变量

传统动态组态通常采用全局变量列表模式

例如：

Motor1Running
Motor1Speed
Motor1Alarm

这种方式存在问题：

  • 变量数量巨大
  • 难以组织
  • 难以维护
  • 不符合 PLC 数据结构
  • 通讯映射复杂
  • 引用关系混乱

因此当前系统采用结构体变量组织数据

例如：

Motor1
├── Running
├── Speed
├── Alarm
└── Temperature

这样更符合

  • PLC 数据块
  • 设备结构
  • 工业对象模型
  • 通讯映射方式

字段模型

字段模型用于描述结构体中的具体数据点

典型结构：

DynamicViewVariableFieldModel
├── FieldId
├── Name
├── VariableType
├── DefaultValue
├── FixedString
├── Description
└── Properties

其中：

属性 说明
FieldId 字段唯一标识
Name 字段名称
VariableType 字段数据类型
DefaultValue 字段默认值，运行时按 VariableType 转换
Description 字段描述
FixedString 固定字符串配置，仅字符串类型使用
Properties 扩展属性

访问模式

访问模式定义在结构体变量层。当前结构体变量整体共享访问模式，而不是每个字段单独配置访问模式

访问模式主要用于：

  • 变量选择过滤
  • 控件绑定限制
  • 通讯读写限制
  • 写回限制
  • 后续权限控制

常见模式：

  • ReadOnly
  • WriteOnly
  • ReadWrite

数据类型

字段通过VariableType描述数据类型，当前常见类型包括：

  • Bool
  • Int16
  • UInt16
  • Int32
  • UInt32
  • Float
  • Double
  • String

字段类型主要用于：

  • 默认值转换
  • 变量选择过滤
  • 控件绑定适配
  • 通讯数据解析
  • 通讯数据写回
  • 运行时值转换

变量筛选

变量选择场景中，通常需要支持筛选

常见筛选条件包括：

  • 分组
  • 数据类型
  • 访问模式
  • 启用状态
  • 关键字

筛选主要用于：

  • 控件绑定
  • 通讯通道绑定
  • 变量监控
  • 事件配置
  • 脚本变量选择

批量更新

变量系统需要支持批量更新

原因是 PLC 通讯通常不是逐字段读取，而是按设备、通道或结构体批量读取

批量更新可以减少：

  • 通知次数
  • UI 刷新次数
  • Dispatcher 压力
  • 高频轮询开销

推荐运行时更新链路：

CommunicationService
    ↓
VariableService
    ↓
RuntimeBindingManager
    ↓
WPF Control

变量监控

变量监控用于观察运行时变量状态

通常显示：

  • 结构体名称
  • 字段名称
  • 数据类型
  • 当前值
  • 默认值
  • 访问模式
  • 启用状态
  • 更新时间

变量监控应读取运行时变量服务中的当前值，而不是直接依赖字段定义模型

引用追踪

变量系统需要支持引用追踪，引用追踪用于分析：变量字段被哪里使用

常见引用来源：

  • 控件绑定
  • 通讯通道
  • 事件配置
  • 后续报警配置
  • 后续趋势配置

引用追踪主要用于：

  • 删除检查
  • 重命名影响分析
  • 工程维护
  • 调试排查
  • 无效引用检测

克隆规则

变量模型和字段模型支持克隆

当前克隆方式基于 JSON 序列化和反序列化

克隆主要用于：

  • 编辑副本
  • 弹窗编辑
  • 撤销重做
  • 临时修改
  • 深拷贝模型

注意：克隆结果会复制当前对象数据，但应避免把运行时缓存、服务实例或临时状态放入变量定义模型中
