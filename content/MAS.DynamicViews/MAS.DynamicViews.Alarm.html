报警系统

报警系统用于把 PLC 或离线模拟产生的 Bool 状态转换为当前报警和历史报警记录。报警点以专用结构体管理，报警列表控件按结构体整体绑定，不进行字段级绑定。

功能组成

报警链路由以下部分组成：

报警结构体
    ↓
设备通道整体轮询 / 离线变量写入
    ↓
变量中心 ValueChanged
    ↓
报警状态跟踪器
    ├── false → true：触发报警
    └── true → false：恢复报警
    ↓
当前报警列表 + AlarmRecords 历史记录

创建报警结构体

在数据中心的“添加结构体”菜单中选择“报警结构体”，输入报警点数量后确认。报警点数量范围为 1 到 1000，默认值为 100

系统生成的结构体具有以下规则：

  • Usage 为 Alarm
  • 访问模式默认为 ReadOnly
  • 字段名依次为 Alarm001、Alarm002……AlarmN
  • 所有字段必须为 Bool，默认值为 false
  • 字段描述作为报警信息；描述为空时使用字段名
  • 字段报警等级默认为“警告”
  • 报警等级支持“提示”“警告”“错误”

数据中心只在报警结构体的字段编辑区显示报警等级。执行检查时，如果报警结构体包含非 Bool 字段，检查不会通过

配置通讯通道

报警结构体可以像普通结构体一样在设备中心关联通道。由于字段全部为 Bool，Modbus TCP 通道应使用 Coils 或 DiscreteInputs 地址区，并保证起始地址和字段顺序与设备数据布局一致

在线运行时，轮询线程读取整个报警结构体并提交变量中心。报警结构体是只读结构体，不参与设备写回

添加并绑定报警列表

  1. 在设计器工具箱的“报警”分类中拖入报警列表控件
  2. 在绑定检查器的“报警绑定”区域启用绑定
  3. 选择一个报警结构体。选择器只显示 Usage=Alarm 的结构体
  4. 检查并同步工程数据，然后打开运行视图

报警列表包含两个页签：

  • 当前报警：显示触发时间、带颜色区分的报警等级和报警信息
  • 历史记录：显示触发时间、恢复时间、报警等级和报警信息

历史记录只查询最近 3 个月的数据，并使用分页控件按触发时间倒序显示

触发与恢复语义

报警状态跟踪器只记录有效跳变：

旧值 新值 结果
false true 新增当前报警，并写入一条未恢复的历史记录
true true 不重复触发
true false 从当前报警移除，并更新历史记录的恢复时间
false false 无操作

通讯整体读取和变量监控的本地写入都通过变量中心的同一变化事件进入报警检测。数据库操作串行执行，以保证同一个实时跳变序列中先插入触发记录、再更新恢复记录

离线模拟

报警结构体默认只读。为了便于无设备调试，离线模式对报警结构体提供一个受限特例：可以在变量监控中把报警点写为 true 或 false，写入只提交到本地变量中心

在线模式仍严格遵守只读限制，不允许通过变量写入服务向报警结构体写值

推荐的离线验证步骤：

  1. 打开运行视图并确认当前为离线模式
  2. 在变量监控中把某个报警点从 false 改为 true
  3. 确认当前报警页出现对应记录和等级颜色
  4. 把该报警点改回 false，确认它从当前报警页消失
  5. 数据库可用时，切换到历史页确认触发时间和恢复时间

数据库记录

报警历史保存在自动创建的 AlarmRecords 表中，主要字段如下：

字段 说明
StructureId / StructureName 报警结构体标识和触发时的名称
FieldId / FieldName 报警点标识和触发时的名称
Level 触发时的报警等级
Description 触发时的报警信息
ActivatedAt 触发时间
ResolvedAt 恢复时间；为空表示尚未恢复

表上为结构体标识和触发时间建立索引。历史页以 StructureId 过滤，因此结构体改名不会丢失已有历史；删除并重建结构体会产生新的标识，不会自动关联旧记录

绑定限制

报警结构体与配方结构体都属于特殊用途结构体，不能在文本、状态、数值等普通控件的数据绑定选择器中选择。报警列表使用专用的结构体整体绑定

事件系统的“变量操作”仍保留原有的特殊结构体选择行为，以兼容配方字段操作；报警结构体因为访问模式为只读，不会出现在变量操作的字段选择结果中

当前边界

报警服务启动或结构体定义变化时，会根据变量中心的当前值重建“当前报警”列表。这个重建过程不会补写历史记录。因此，如果报警点在服务启动前已经是 true，运行视图会显示当前报警，但历史表不会新增一次触发记录；历史记录只记录服务运行期间观察到的实时跳变

重建时无法得知报警的真实触发时刻，当前报警页会把本次扫描时间作为显示时间。如果上一运行周期留下了未恢复的历史记录，而报警在程序停止期间已经恢复，本次启动只观察到初始 false，不会自动补写该历史记录的恢复时间

历史查询的取消令牌只在查询开始前检查，底层仓储查询开始后不会中断。快速翻页时，控件通过查询版本号丢弃过期结果，避免旧页面覆盖新页面

排查清单

当前报警未出现时，依次检查：

  • 结构体 Usage 是否为 Alarm
  • 报警字段是否为 Bool，结构体是否已启用
  • 报警列表是否已启用绑定并选择了正确的结构体
  • 在线模式下设备、通道、地址和结构体绑定是否有效
  • 修改后是否执行了检查并同步工程数据
  • 测试值是否确实发生了 false → true 跳变

历史记录未出现时，继续检查数据库连接和 AlarmRecords 表是否可用，并确认触发时间处于最近 3 个月内
