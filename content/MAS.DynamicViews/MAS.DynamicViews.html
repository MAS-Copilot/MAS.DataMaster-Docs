MAS.DynamicViews

动态工业组态与运行时界面引擎简称MAS.DynamicViews

包含设计态编辑器、运行时渲染器、变量系统、通讯绑定、事件系统、控件安全和全局脚本能力，目标是让界面、数据、设备和业务逻辑能够以可配置、可维护、可扩展的方式协同工作

使用场景：

  • 工业设备上位机界面
  • PLC 数据监控
  • 产线状态看板
  • 数据采集界面
  • 多页面工艺流程展示
  • 动态参数配置
  • 设备调试工具
  • 简易 SCADA/HMI 系统
  • 可视化运行监控平台

特性

  • 动态页面设计器
  • 设计态与运行态分离
  • MVVM 数据绑定
  • PLC 通讯集成
  • Modbus TCP 通讯支持
  • 通讯轮询与变量写回
  • 运行时事件系统
  • 控件预设系统
  • 页面缩放与布局系统
  • 多页面工程
  • 工程持久化
  • Undo/Redo 历史系统
  • 自定义控件扩展
  • 结构体变量绑定
  • 字段级变量绑定
  • 报警结构体、当前报警与历史报警
  • 变量引用追踪
  • 设备中心
  • 数据中心
  • 变量监控
  • 全局脚本
  • 控件权限、长按、防抖、二次确认和操作日志
  • 页面、窗口和背景内容
  • 主题与外观配置
  • 控件属性检查器
  • 控件事件检查器
  • 拖拽式控件创建
  • 控件选择、移动、缩放与对齐辅助
  • 运行时宿主集成
  • 后续插件化扩展能力

核心模块

工程系统

工程系统负责维护动态视图项目的整体数据，包括页面、变量、设备、参数、启动页、页面尺寸等信息

主要职责：

  • 管理项目基础信息
  • 管理多页面数据
  • 管理项目级页面尺寸
  • 管理启动页
  • 管理变量、设备、通讯配置
  • 支持项目持久化与恢复

页面、窗口与背景系统

页面系统负责描述主运行区域中的动态界面。窗口用于在运行时弹出独立操作界面，背景用于复用页眉、页脚、导航栏、状态栏等公共内容

主要职责：

  • 页面尺寸管理
  • 页面背景与外观配置
  • 页面控件集合管理
  • 页面切换
  • 页面缩放
  • 页面控件同步缩放
  • 运行时打开窗口
  • 公共背景复用

运行时默认显示启动页；如果未配置启动页，则显示工程中的第一个页面。打开窗口事件会根据窗口配置决定是否允许重复打开、是否以模态方式显示、是否允许调整大小

控件系统

控件系统负责描述动态界面中的所有控件

每个控件通常包含：

  • 基础信息
  • 布局属性
  • 外观属性
  • 文本属性
  • 数据绑定
  • 事件配置

控件本身不直接依赖 WPF 控件实例，而是通过模型描述，再由渲染器生成真实界面元素

预设系统

预设系统用于快速创建常用控件

主要职责：

  • 注册内置控件预设
  • 按分类组织控件
  • 支持拖拽创建控件
  • 支持扩展自定义预设
  • 统一控件默认属性和绑定配置

当前内置预设包括：

  • 按钮类：普通按钮、切换按钮、开关按钮、单选按钮、复选框、卡片动作按钮
  • 文本类：文本块、文本框
  • 装饰类：边框、卡片、分组框
  • 显示类：图片、图标、进度条、赛博朋克进度条
  • 指示器类：双色呼吸灯、双色常亮灯、数值状态灯

设计器还支持绘图方式创建矩形、圆角矩形、椭圆、直线、折线、箭头、多边形、贝塞尔曲线和圆弧。矩形类、直线和箭头通过拖拽创建；折线、多边形和贝塞尔曲线通过逐点点击创建，双击、回车或右键结束，Esc 取消

设计器系统

设计器系统负责动态界面的可视化编辑

主要职责：

  • 控件渲染
  • 控件拖拽
  • 控件选择
  • 控件移动
  • 控件缩放
  • 多选操作
  • 对齐辅助
  • 网格背景
  • 缩放视图
  • 属性编辑
  • 事件编辑
  • 历史记录

设计器中的控件通过 DesignerControlChrome 包装，选择框、缩放点等设计态元素不会直接改变控件本体尺寸，避免设计态视觉元素影响真实布局

渲染系统

渲染系统负责将动态视图模型转换为 WPF 界面元素

当前主要分为：

  • 设计态渲染器
  • 运行态渲染器

设计态渲染器侧重编辑体验，运行态渲染器侧重数据绑定、事件执行和业务交互

运行时系统

运行时系统负责项目运行时的页面展示、变量绑定、事件执行和宿主交互

主要职责：

  • 加载运行页面
  • 渲染真实控件
  • 建立变量绑定
  • 绑定运行事件
  • 响应用户操作
  • 与变量服务、通讯服务、宿主服务协同

变量系统

变量系统是动态视图的数据核心。

当前变量系统以结构体变量为主要方向，支持结构体字段级绑定

主要职责：

  • 管理结构体变量
  • 管理字段定义
  • 管理字段值
  • 支持变量监控
  • 支持变量更新通知
  • 支持结构体整体更新
  • 支持字段级更新
  • 支持批量更新

绑定系统

绑定系统负责将控件属性和变量字段建立关联

常见绑定类型包括：

  • 文本绑定
  • 可见性绑定
  • 启用状态绑定
  • 颜色绑定
  • 选中状态绑定
  • 状态绑定
  • 数值状态绑定
  • 数值绑定

绑定目标采用结构体字段路径，例如：

结构体.字段

绑定配置内部保存结构体标识和字段标识，显示名称仅用于 UI 展示。绑定必须启用并选择有效字段后才会在运行时生效

典型运行效果：

  • 文本绑定可显示变量值，并支持格式化和小数位换算
  • 文本框在失去焦点时写回绑定字段；按 Enter 会先移出焦点，再触发写回
  • 开关、复选框、单选按钮等选中状态绑定会在用户切换时写回布尔值；写入失败时界面会恢复为当前变量值
  • 可见性、启用状态、颜色和状态灯绑定通常使用布尔字段
  • 数值状态灯根据数值到颜色的映射显示状态，未匹配时使用备用颜色
  • 进度条类控件使用数值绑定，变量值会被限制在控件最小值和最大值范围内

通讯系统

通讯系统负责设备数据和动态视图变量之间的数据同步

主要职责：

  • 管理设备配置
  • 管理通讯通道
  • 支持 Modbus TCP
  • 支持通道读取
  • 支持变量写回
  • 支持轮询更新
  • 支持结构体字段映射
  • 支持运行期缓存，避免频繁读取持久化工程数据

通讯系统通常包含：

  • 设备中心
  • 通道配置
  • 轮询更新服务
  • 写回服务
  • 通讯服务
  • 协议适配层

数据中心

数据中心负责维护动态视图中的结构体变量、字段、分组、访问模式和数据类型

除普通结构体和配方结构体外，数据中心还支持创建只读 Bool 报警结构体，并为每个报警点配置报警等级和描述。报警结构体的完整使用流程参见报警系统

主要职责：

  • 结构体变量管理
  • 字段管理
  • 数据类型配置
  • 访问模式配置
  • 默认值配置
  • 变量校验
  • 变量选择
  • 变量引用查询

设备中心

设备中心负责维护外部设备与变量之间的关联

主要职责：

  • 设备配置
  • 通讯协议配置
  • 通道配置
  • 地址配置
  • 数据区配置
  • 结构体字段绑定
  • 读写权限约束
  • 通讯参数配置

当前通讯协议支持 Modbus TCP。设备中心中的改动需要通过“同步工程数据到内存”写入工程运行数据；未同步的改动不会参与运行时通讯

Modbus 通道配置时需要注意：

  • 线圈区和保持寄存器支持写入
  • 离散输入区和输入寄存器只读
  • 线圈区、离散输入区适合绑定 Bool 字段
  • 保持寄存器、输入寄存器适合绑定 Int16 或 UInt16 字段
  • 通道地址必须是有效的无符号短整型地址
  • 只写结构体只能绑定到可写数据区

引用追踪系统

引用追踪系统用于查看变量、字段、设备通道、控件绑定和事件配置之间的引用关系

主要目标：

  • 查找某个变量被哪些控件绑定
  • 查找某个字段被哪些设备通道关联
  • 查找某个变量是否被事件或脚本引用
  • 删除变量前进行安全检查
  • 降低后期维护和排查成本

事件系统

事件系统用于描述控件在运行时触发的业务行为

当前控件事件包括：

  • 点击
  • 按下
  • 松开

每个事件可配置以下动作：

  • 切换页面
  • 打开窗口
  • 变量操作

变量操作支持：

  • 设置为 0 或 False
  • 设置为 1 或 True
  • 取反
  • 递增
  • 递减

一个事件中可以同时启用多个动作。运行时会先执行变量操作，再执行页面跳转和打开窗口

典型使用方式：

  • 点击模式：适用于启动、停止、复位、页面跳转等一次性操作
  • 按下模式：适用于点动、手动夹紧、按住才允许动作的控制
  • 松开模式：适用于释放点动、撤销保持信号、结束临时控制

事件系统应与运行时上下文协同，避免设计态与运行态逻辑混杂

控件安全

控件安全用于保护运行时操作，适合设备启停、手动动作、配方下发等风险较高的按钮

可配置能力：

  • 最低操作角色：Visitor、Operator、Engineer、Admin
  • 长按时间：仅作用于按下事件
  • 防抖：避免短时间重复触发按下事件
  • 二次确认：执行前弹出确认窗口
  • 操作日志：记录操作者、动作、资源和耗时

注意：权限、长按、防抖和二次确认只在事件中存在已启用动作时才会执行。长按、防抖和二次确认属于按下事件保护；点击和松开事件仍会做权限校验和操作日志

脚本系统

脚本系统用于提供更灵活的业务扩展能力

当前只支持全局脚本，不区分页面脚本、窗口脚本或控件事件脚本

全局脚本适合：

  • 监听变量变化
  • 监听页面导航
  • 编写工程级公共逻辑
  • 通过公开服务读取或写入变量
  • 调用页面导航能力
  • 访问动态视图管理能力

脚本系统应受到权限、安全和运行时上下文约束，避免影响系统稳定性

历史系统

历史系统用于支持设计器中的撤销和重做

主要职责：

  • 添加控件历史
  • 删除控件历史
  • 移动控件历史
  • 属性修改历史
  • 页面操作历史
  • 限制最大历史数量

持久化系统

持久化系统负责将动态视图工程保存到项目数据中，并在需要时恢复

主要职责：

  • 保存工程模型
  • 恢复工程模型
  • 标记工程脏状态
  • 监听变量变化
  • 监听页面变化
  • 与主工程系统集成
