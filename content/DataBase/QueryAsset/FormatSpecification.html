查询预设 MD 格式规范

纯 Markdown，表格 + 代码块都是 Markdown 原生语法，可无损转换为 HTML / Word / PDF

一、文件与层级

  • 一个 .md 文件 = 一个一级分类，分类名默认取文件名（去掉扩展名）。如需自定义显示名，可在文件开头写一个 # 一级标题覆盖它。
  • 文件内每个 ## 二级标题小节 = 一条查询（二级）。
  • 不含 SQL 代码块的小节会被自动忽略（例如「使用说明」「表结构说明」），可以放任意文档说明，不影响解析

二、单条查询的结构（固定顺序）

## 查询标题                         ← 1. H2，作为查询标题

一句话描述放在这里。                 ← 2. 描述（紧跟标题的第一段文字，可省略）

| 参数 | 标签 | 类型 | 默认值 | 是否可空 | 说明 |   ← 3. 参数表（无参数则整张表省略）
| --- | --- | --- | --- | --- | --- |
| ProductId | 产品 ID | text | PROD-001 | false | 要查询的产品编号 |

​```sql                             ← 4. SQL 代码块，用 {{参数名}} 占位
SELECT ... WHERE d.ProductId = {{ProductId}};
​```

三、参数表列定义（6 列，「是否可空」列可省略）

列 含义 说明
参数 占位符名 对应 SQL 里的 {{参数名}}，英文、唯一
标签 界面显示名 表单里那个中文 label
类型 控件类型 text / number / date / datetime / enum，详见「五、类型 → 控件映射」
默认值 预填值 打开查询时输入框的初始值
是否可空 允许留空 true / false，留空视为 false；兼容旧格式：整列省略（5 列表）时全部参数默认不可空。详见「七、可空参数」
说明 输入框提示 显示在控件下方的灰字，可留空；enum 类型在此列声明候选项（见第六节）

四、占位符与参数替换

  • SQL 里统一用 {{参数名}} 双大括号占位
  • 替换规则按类型决定要不要加引号：
    • text / date / datetime / enum → 自动加单引号，如 'PROD-001'、'2026-06-01 00:00:00'、'OK'
    • number → 不加引号，如 100
  • 兜底：如果 SQL 里出现了某个 {{X}}，但参数表里没列它，就当作 text 类型、标签 = X

五、类型 → 控件映射

类型 渲染的控件 备注
text 文本输入框 自由输入文本
number 数值输入框 仅接受数值，替换时不加引号
date 日期选择器（DatePicker） 选日期（不含时间），写回 yyyy-MM-dd
datetime 文本框 手动输入完整日期时间，如 2026-06-01 00:00:00
enum 下拉框（ComboBox） 候选项在「说明」列声明，见第六节

六、enum（下拉框）

enum 用于把参数限定为若干预设值，渲染为下拉框

  • 声明候选项：在「类型」列写 enum，候选项写在「说明」列，格式 选项: A,B,C
    • 冒号支持中/英文（: 或 ：），逗号支持中/英文（, 或 ，）
    • 冒号前的文字（如「选项」）只是提示词，不参与解析；冒号后按逗号切分得到候选项
  • 值始终按字符串处理：选中项替换进 SQL 时会加单引号（如 'OK'）。即便候选项看起来像数字（选项: 1,2,3），替换结果仍是字符串 '1'。如需不加引号的数值，请用 number 类型
  • 默认值：若想打开时预选某项，「默认值」列必须与某个候选项完全一致（区分大小写）
  • 提示文字：整串「说明」（含 选项: 前缀）会作为灰字提示显示在下拉框下方

示例参数行：

参数 标签 类型 默认值 是否可空 说明
Result 判定结果 enum OK false 选项: OK,NG,未知

七、可空参数（是否可空）

「是否可空」列声明参数是否允许留空，用于「留空 = 查询全部」这类场景

  • 取值：true（可空）/ false（不可空），留空视为 false；也兼容中文 是
  • 兼容旧格式：没有「是否可空」列的 5 列参数表仍然有效，所有参数默认不可空
  • 校验行为：不可空参数留空会阻止执行并提示「不能为空」；可空参数留空则跳过校验直接执行
  • 留空时的替换规则（与预览一致）：
    • text / date / datetime / enum → 空字符串 ''
    • number → NULL
  • SQL 写法：留空查询全部需要在 SQL 里自行兜底，例如：
-- text：留空时 {{ProductId}} = ''，条件恒真，返回全部
WHERE ({{ProductId}} = '' OR f.ProductId = {{ProductId}})

-- number：留空时 {{Limit}} 为 NULL
WHERE ({{Limit}} IS NULL OR f.Thickness < {{Limit}})

示例参数行：

参数 标签 类型 默认值 是否可空 说明
ProductId 产品 ID text true 留空查询全部产品
