查询预设/资产功能

概述

用于将常用数据库查询封装为可复用的查询资产。可以提前编写并导入标准化的 Markdown 查询预设文件，系统解析后自动生成查询分类、查询卡片、参数输入区和查询结果区。用户无需复制 SQL、修改 SQL 或理解数据库结构，只需要选择查询功能并填写业务参数，即可完成数据查询

该功能的核心目标是将“写 SQL 查询”转变为“使用查询功能”，降低现场操作复杂度，提高查询效率，并减少因手动修改 SQL 导致的错误

解析逻辑

整个解析就是"切块 → 判断 → 抽字段"五步:

  1. 定分类:读入一个.md,文件名去扩展名 = 一级分类名(也可被文件开头的 # 标题覆盖)
  2. 按##切块:用^##把正文切成若干小节,每个小节是一条候选查询
  3. 判断是不是查询:看这个小节里有没有 ```sql 代码块。没有就跳过——这是"使用说明""表结构说明"这类文档段被自动忽略的原因
  4. 抽四个字段(对有 SQL 的小节):
    • 标题 = ##后面的文字
    • 描述 = 标题和(表格/代码块)之间的第一段普通文字
    • 参数 = 解析那张 Markdown 表格,逐行变成{name, label, type, default, hint}(跳过 --- 分隔行)
    • SQL = 代码块里的内容
  5. 校验 + 产出:用/\{\{(\w+)\}\}/把 SQL 里所有占位符扫出来,跟参数表对一遍(缺的兜底成 text)。最终每个文件产出{ 分类名, queries: [{id, title, description, params, sql}] }

关键正则就三条:切小节/^## /m、取 SQL/```sql\s*([\s\S]*?)```/、取占位符/\{\{(\w+)\}\}/g。表格按行 split |即可
