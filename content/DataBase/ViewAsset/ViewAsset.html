ViewAsset 视图资产

模块说明

ViewAsset 用于承载自定义的数据库视图页面。每个视图资产是项目资产根目录下的独立文件夹，由普通 HTML、CSS、JavaScript、图片以及只读 SQL 文件组成。

DataMaster 启动本地 Web Host，负责：

  • 托管资产目录中的前端文件；
  • 提供只读 SQL 查询接口；
  • 使用当前 ViewAsset 项目配置的数据库执行查询；
  • 将查询结果转换为 JSON 返回给资产页面。

前端页面不直接连接数据库。页面只能通过 ViewAsset 查询接口请求后端执行资产目录中的 SQL 文件。

ViewAsset 不负责生成页面中的查询控件。文本框、日期选择器、查询按钮、数据表格、图表和刷新逻辑都由资产自己的前端代码实现。

相关模块

MAS.Database/ViewAsset

该模块实现 ViewAsset 的核心业务逻辑：

  • 资产索引与目录解析；
  • manifest.json 读取；
  • 资产文件扫描与校验；
  • SQL 文件定位；
  • 只读 SQL 校验；
  • SQL 参数命令构建；
  • 数据库查询执行；
  • 查询结果转换；
  • ViewAsset 项目持久化。

查询相关的主要类型如下：

  • ViewAssetQueryRequest：查询请求模型；
  • ViewAssetQueryRequestParser：HTTP 与 IPC 共用的请求与参数解析、类型转换和校验；
  • IViewAssetQueryService：查询服务接口；
  • ViewAssetQueryService：SQL 文件读取、校验和执行；
  • ViewAssetSqlCommandBuilder：占位符转换、缺参检查和参数绑定命令构建；
  • ViewAssetSqlGuard：单条只读 SQL 校验；
  • ViewAssetQueryException：数据库执行阶段失败（消息含 SQL 文件名，已脱敏）；
  • ViewAssetResultConverter：查询结果转换；
  • ViewAssetQueryResult：页面查询结果；
  • ViewAssetPreviewResult：工具预览结果。

MAS.DataMaster/Services/ViewAsset

ViewAssetWebModule 将 ViewAsset 接入本地 Web Host，处理静态文件请求与页面查询请求。

MAS.DataMaster/NamedPipeIpc/ViewAsset

ViewAssetToolProvider 向 PrismaTalk/IPC 暴露 viewAsset.* 工具，包括资产查询、校验、注册、注销、恢复、预览、运行验证和封面生成。

其中 viewAsset.preview 调用的也是 IViewAssetQueryService，并且与页面 HTTP 查询共用同一个 ViewAssetQueryRequestParser 完成请求解析、参数类型转换与校验，两条入口的参数语义完全一致。

资产目录

典型资产结构：

ViewAssets/
├─ _shared/
│  └─ echarts.min.js
└─ production-query/
   ├─ index.html
   ├─ manifest.json
   ├─ app.js
   ├─ style.css
   └─ main.sql
  • index.html：默认入口页面；
  • manifest.json：资产描述文件；
  • *.sql：可由页面查询接口执行的只读 SQL；
  • _shared：所有 ViewAsset 可引用的共享前端资源目录。

manifest.json 当前描述资产入口、标题、说明、标签、样式、组件数量和刷新策略，不包含查询参数定义。

Web Host 路由

静态页面

GET /modules/view-assets/{assetId}/
GET /modules/view-assets/{assetId}/{relativePath}

空路径加载 manifest.entry 指定的入口文件；未配置时使用 index.html。其他路径用于加载资产目录内的 CSS、JavaScript、图片等静态文件。

共享资源通过以下路径访问：

GET /modules/view-assets/_shared/{fileName}

查询接口

POST /api/view-assets/{assetId}/query
Content-Type: application/json

查询接口只支持 POST。其他 HTTP 方法返回 405；不支持的 API 动作返回 404。

页面查询请求

请求结构：

{
  "sqlFile": "main.sql",
  "parameters": {
    "Keyword": "%电机%",
    "MinQuantity": 10,
    "Enabled": true,
    "LineId": null,
    "StartTime": { "type": "datetime", "value": "2026-07-01T00:00:00+08:00" }
  },
  "rowLimit": 5000
}

字段说明：

字段 类型 必填 说明
sqlFile string 是 资产目录内 SQL 文件的相对路径，必须以 .sql 结尾
parameters object 否 参数名到参数值的映射，键名不区分大小写；值保留 JSON 原生类型，也支持显式类型包装
rowLimit number 否 返回行数上限；小于等于 0 时使用默认值 5000

请求体为空、JSON 无效、字段类型错误或缺少 SQL 参数时返回 HTTP 400，错误信息会说明具体原因。

页面调用示例：

async function runQuery(parameters) {
  const response = await fetch(`/api/view-assets/${assetId}/query`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      sqlFile: "main.sql",
      parameters,
      rowLimit: 5000
    })
  });

  const result = await response.json();
  if (!response.ok) {
    throw new Error(result.error || `HTTP ${response.status}`);
  }

  return result;
}

SQL 参数处理

占位符格式

当前 SQL 文件使用 {{参数名}} 占位符：

SELECT id, product_name, created_at
FROM production_record
WHERE product_name LIKE {{Keyword}}
  AND created_at >= {{StartTime}}
  AND created_at < {{EndTime}}
ORDER BY created_at DESC

ViewAssetSqlCommandBuilder 从 SQL 中提取占位符，并将其转换为数据库命名参数：

SELECT id, product_name, created_at
FROM production_record
WHERE product_name LIKE @Keyword
  AND created_at >= @StartTime
  AND created_at < @EndTime
ORDER BY created_at DESC

参数值保存在 PreparedSqlCommand.Parameters 中。QueryExecutor 创建数据库命令后，通过 AddParameter 将每个值绑定到对应的 @参数名。

因此，当前实现不是把参数值直接替换或拼接到 SQL 文本中。SQL 结构来自资产目录中的 SQL 文件，前端只提交参数值。

参数语义

ViewAsset 没有参数定义模型，前端完全自由地决定发送哪些参数。ViewAssetQueryRequestParser 解析请求后，ViewAssetQueryRequest.Parameters 的类型为：

IReadOnlyDictionary<string, object?>

解析与绑定遵循以下规则：

  • 缺失参数直接报错：SQL 使用的每个 {{占位符}} 都必须在 parameters 中出现，缺失时返回 HTTP 400 并列出缺失的参数名，例如 SQL 文件「main.sql」缺少参数：StartTime，不再静默绑定空字符串；
  • 保留 JSON 原生类型：字符串按文本绑定（不做 Trim，按原样传入数据库）、数字按数字绑定（整数转 long，小数转 decimal）、布尔按布尔绑定；
  • null 绑定数据库 NULL：显式传 null 时按 DBNull 绑定，可配合 ({{LineId}} IS NULL OR line_id = {{LineId}}) 实现可选条件；
  • 空字符串绑定空字符串："" 与 null 不再混同；
  • 多余参数忽略：请求中存在但 SQL 未使用的参数不会加入数据库命令；
  • 数组值报错：parameters 的值不支持数组。

参数名称通过正则表达式 {{(\w+)}} 提取并不区分大小写去重，同名占位符只生成一个绑定参数；大小写不同的同名占位符（如 {{Start}} 与 {{START}}）会替换为同一个命名参数。

显式类型参数

普通字符串永远按文本绑定，不做任何自动类型猜测。需要让数据库拿到真正的日期或数字类型时，使用显式类型包装：

{
  "parameters": {
    "StartTime": { "type": "datetime", "value": "2026-07-01T00:00:00+08:00" },
    "MinQuantity": { "type": "number", "value": "10" }
  }
}

type 支持以下取值（value 为 null 时一律绑定数据库 NULL）：

type value 要求 绑定结果
text 字符串 文本
number 数字或数字字符串（不变量文化） long / decimal / double
boolean 布尔或 "true"/"false" 字符串 bool
date ISO 8601 字符串 DateTime，只保留日期部分
datetime ISO 8601 字符串 DateTime
json 任意 JSON 原始 JSON 文本（按文本绑定）

日期时间的时区规则：带时区偏移（含 Z）的值会换算为 DataMaster 所在机器的本机时间后绑定；不带时区的值按本机时间原样绑定。type 拼写错误或 value 无法转换都会返回明确错误。

可选条件与 NULL

参数只能代表「值」，不能改变 SQL 结构。动态启用/停用条件推荐用 NULL 判断写在 SQL 里：

SELECT id, product_name, created_at
FROM production_record
WHERE ({{Keyword}} IS NULL OR product_name LIKE {{Keyword}})
  AND ({{StartTime}} IS NULL OR created_at >= {{StartTime}})

前端对「不过滤」的条件显式传 null：

{
  "sqlFile": "main.sql",
  "parameters": { "Keyword": "%电机%", "StartTime": null }
}

查询形态差异较大时（如不同排序、不同聚合），更适合拆成多个 SQL 文件由前端选择，而不是把 SQL 结构做成参数。

模糊查询

模糊查询由 SQL 的 LIKE 条件与前端传入的通配符共同完成：

SELECT id, product_name
FROM production_record
WHERE product_name LIKE {{Keyword}}

前端包含匹配：

await runQuery({
  Keyword: `%${keyword.trim()}%`
});

前端也可以根据查询需求传入：

  • %关键字%：包含匹配；
  • 关键字%：前缀匹配；
  • %关键字：后缀匹配。

值最终作为 @Keyword 的参数绑定，不会直接拼入 SQL。

日期时间查询

日期输入同样由资产前端负责。页面可以使用 input[type=date]、input[type=datetime-local] 或其他日期组件产生字符串，然后作为参数提交：

await runQuery({
  StartTime: "2026-07-01 00:00:00",
  EndTime: "2026-08-01 00:00:00"
});

SELECT id, created_at
FROM production_record
WHERE created_at >= {{StartTime}}
  AND created_at < {{EndTime}}

普通字符串仍按文本绑定，不会自动转换为 .NET DateTime；具体数据库是否接受该字符串格式，以及比较时如何进行隐式类型转换，由数据库和驱动决定。需要跨数据库稳定的日期比较时，改用显式类型包装：

await runQuery({
  StartTime: { type: "datetime", value: "2026-07-01T00:00:00+08:00" },
  EndTime: { type: "datetime", value: "2026-08-01T00:00:00+08:00" }
});

推荐使用 ISO 8601 并显式携带时区偏移；不带时区的值按 DataMaster 本机时间处理。

日期范围通常使用左闭右开条件。例如查询 2026 年 7 月时，开始值为 2026-07-01 00:00:00，结束值为 2026-08-01 00:00:00。

查询执行流程

ViewAssetQueryService.ExecuteAsync 的执行顺序如下：

  1. 使用资产 ID 或名称查找资产；
  2. 拒绝不存在或已注销的资产；
  3. 解析资产实际目录；
  4. 校验 sqlFile 非空并以 .sql 结尾；
  5. 将相对路径解析到资产目录内；
  6. 拒绝路径穿越和不存在的文件；
  7. 异步读取 SQL 文件文本；
  8. 调用 ViewAssetSqlGuard.TryCheckReadonlySql；
  9. 调用 ViewAssetSqlCommandBuilder.Build 构建参数化命令（SQL 使用的占位符缺少参数时在此报错）；
  10. 从当前 ViewAsset 项目取得数据库配置；
  11. 将数据库实例设置到 IQueryExecutor；
  12. 调用 IQueryExecutor.ExecuteAsync 执行查询（执行失败包装为 ViewAssetQueryException，消息含 SQL 文件名并脱敏）；
  13. 将 DataTable 转换为 ViewAsset 查询结果。

当前查询没有前端 SQL 文本入口。后端执行的 SQL 必须来自资产目录中的实际 .sql 文件。

SQL 文件与路径限制

sqlFile 只能是资产目录内的 .sql 文件：

  • 不接受 index.html 等其他扩展名；
  • 不接受不存在的文件；
  • 不允许通过 ../ 离开资产目录；
  • 不允许请求直接传递 SQL 文本。

SQL 文件选择可以由前端控制。例如一个资产可以包含 summary.sql、detail.sql 和 trend.sql，页面根据功能调用不同文件，但每个文件都必须已经存在于当前资产目录内。

只读 SQL 校验

ViewAssetSqlGuard 在执行前移除行注释与块注释，然后进行以下检查：

  1. SQL 内容不能为空；
  2. 去掉末尾分号后，内容仍不能为空；
  3. SQL 中不能存在其他分号，即只能执行一条语句；
  4. 首关键字只能是：
    • SELECT
    • WITH
    • SHOW
    • DESCRIBE
    • DESC
    • EXPLAIN
  5. 对 SELECT 和 WITH 检查禁止关键字，包括 INSERT、UPDATE、DELETE、DROP、ALTER、CREATE、TRUNCATE、GRANT、REVOKE、REPLACE、MERGE、CALL、EXECUTE、EXEC、INTO、LOAD、HANDLER、COPY 和 VACUUM。

校验失败时不会创建数据库查询，接口返回 HTTP 400 和具体错误信息。

行数限制

页面查询的默认行数限制为 5000：

  • rowLimit > 0：使用请求值；
  • rowLimit <= 0：使用默认值 5000。

QueryExecutor 在执行前根据数据库类型调用 DbQueryHelper.ApplyRowLimit，将行数限制应用到 SQL。

viewAsset.preview 还会对返回给工具调用方的 records 再做一次预览截断。默认只返回前 50 条记录，最多允许请求 200 条；数据库查询本身仍使用 rowLimit。

页面查询响应

成功响应：

{
  "sqlFile": "main.sql",
  "columns": ["id", "product_name", "created_at"],
  "rows": [
    [1, "电机 A", "2026-07-01 08:30:00"],
    [2, "电机 B", "2026-07-01 09:10:00"]
  ],
  "rowCount": 2,
  "elapsedMilliseconds": 18,
  "executedAt": "2026-07-11T10:00:00"
}

字段说明：

字段 说明
sqlFile 实际执行的 SQL 文件相对路径
columns 按查询结果顺序排列的列名
rows 二维数组，每行单元格顺序与 columns 一致
rowCount 返回结果行数
elapsedMilliseconds 数据库查询耗时，单位毫秒
executedAt 查询完成时间

结果单元格转换规则：

  • DBNull 转换为 null；
  • DateTime 格式化为 yyyy-MM-dd HH:mm:ss；
  • TimeSpan 转换为字符串；
  • byte[] 转换为 null；
  • 其他值保留原始类型。

前端可将二维数组转换为对象数组：

function toRecords(result) {
  return result.rows.map(row =>
    Object.fromEntries(
      result.columns.map((column, index) => [column, row[index]])
    )
  );
}

错误处理

常见响应状态：

状态码 场景
400 JSON 无效、缺少 sqlFile、路径无效、文件不存在、SQL 非只读、缺少 SQL 参数、参数值非法、未配置数据库等
404 资产不存在、资产已注销或 API 动作不存在
405 查询接口使用了非 POST 方法
500 数据库执行失败或服务内部发生未处理异常

错误响应包含 error 字段，前端应同时检查 HTTP 状态和响应内容。错误信息会指明失败阶段与相关名称，且不包含数据库密码等机密：

{ "error": "SQL 文件「main.sql」缺少参数：StartTime（SQL 使用的每个占位符都必须提供参数；显式传 null 表示绑定数据库 NULL）" }

{ "error": "参数「StartTime」的 datetime 值无效：\"昨天\"（要求 ISO 8601 格式，如 2026-07-01、2026-07-01T08:30:00 或带时区 2026-07-01T08:30:00+08:00）" }

{ "error": "SQL 文件「main.sql」执行失败：Unknown column 'produce_date' in 'where clause'" }

数据库执行阶段的错误（HTTP 500）以 SQL 文件「...」执行失败： 开头，底层消息中的 password/pwd 键值会被替换为 ***。

IPC 预览请求

viewAsset.preview 的外层结构与页面 HTTP API 略有不同，parameters 使用数组形式（对象形式也被接受）：

{
  "assetIdOrName": "production-query",
  "sqlFile": "main.sql",
  "parameters": [
    { "name": "Keyword", "value": "%电机%" },
    { "name": "LineId", "value": null },
    { "name": "StartTime", "value": { "type": "datetime", "value": "2026-07-01T00:00:00+08:00" } }
  ],
  "rowLimit": 5000,
  "maxRecords": 50
}

数组元素的 value 与 HTTP 的参数值语义完全相同：保留 JSON 原生类型、null 绑定数据库 NULL、支持显式类型包装。ViewAssetToolProvider 与 ViewAssetWebModule 共用 ViewAssetQueryRequestParser 解析请求，因此缺参报错、类型转换和错误信息在两条入口上保持一致。

预览结果除列、总行数和耗时外，还包含：

  • records：对象形式的前若干条记录；
  • recordsTruncated：是否发生预览截断；
  • recordsTruncatedMessage：截断说明。

当前能力边界

  • 支持一个请求执行一个资产 SQL 文件；
  • 支持字符串、数字、布尔与 null 的 JSON 原生参数，以及 text/number/boolean/date/datetime/json 显式类型包装；
  • 支持通过 LIKE 和前端通配符完成模糊查询；
  • 支持通过 null 参数配合 IS NULL 条件实现可选过滤；
  • 缺少 SQL 参数、参数值非法时明确报错；
  • 不支持数组参数（IN (...) 展开暂未提供）；
  • 不支持请求内联 SQL；
  • 不支持通过参数动态替换表名、列名、排序方向或其他 SQL 结构；
  • 不从 manifest.json 自动生成参数控件；
  • 不包含 ChartAsset 的参数类型定义与默认值合并逻辑；
  • 页面刷新由前端实现，manifest.refresh 仅描述刷新策略，不自动替页面执行查询。
