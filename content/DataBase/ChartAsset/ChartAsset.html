ChartAsset 图表资产

模块说明

ChartAsset 用于把数据库查询、参数定义、仪表盘布局和 ECharts 配置保存为可重复执行的图表资产。一个资产可以表示单个图表，也可以表示由多个图表和摘要卡片组成的仪表盘。

ChartAsset 使用声明式 JSON 模型。后端负责校验资产、合并参数、执行数据源、组装渲染数据；DataMaster 页面负责生成参数输入控件；MAS.Charts 中的 ChartAsset.html 和 ChartAsset.js 负责最终的 HTML、摘要卡片与 ECharts 渲染。

ChartAsset 与 ViewAsset 不同。ChartAsset 不托管每个资产独立编写的 HTML 页面，资产的页面结构由统一的 ChartAsset 前端提供；每个资产主要描述参数、数据源、布局和图表配置。

相关模块

MAS.Database/ChartAsset

该模块实现 ChartAsset 的核心逻辑：

  • 图表资产模型及 JSON 序列化；
  • 资产结构校验；
  • SQL 只读校验；
  • 参数默认值合并和输入校验；
  • SQL 参数化命令构建；
  • SQL 与 C# 脚本数据源执行；
  • 查询结果转换；
  • ECharts option 数据注入；
  • 摘要卡片数据计算；
  • 渲染 Payload 组装；
  • 资产增删改查与项目持久化。

主要服务包括：

  • IChartAssetRuntime / ChartAssetRuntime：执行资产并生成渲染 Payload；
  • IChartAssetToolService / ChartAssetToolService：资产查询、校验、写入、补丁、删除和预览；
  • IChartAssetProjectPersistenceService：资产与数据库配置持久化；
  • IChartAssetScriptRunner / ChartAssetScriptRunner：编译并执行脚本数据源；
  • ChartAssetValidator：完整资产静态校验；
  • ChartAssetSqlGuard：SQL 数据源只读校验；
  • ChartAssetSqlBuilder：参数化 SQL 命令和查询结果转换；
  • ChartOptionComposer：将数据源结果注入 ECharts option；
  • SummaryCardComposer：计算摘要卡片显示数据。

MAS.DataMaster/NamedPipeIpc/ChartAsset

ChartAssetToolProvider 通过 IPC 向 PrismaTalk 暴露 chartAsset.* 工具，用于列出、读取、校验、写入、局部修改、删除和预览资产。

MAS.DataMaster ChartAsset 页面

ChartAssetViewModel 维护资产列表、当前资产、参数值、执行状态和定时刷新。它调用 IChartAssetRuntime 取得 ChartRenderPayload，再通过 ChartAssetWebViewService 推送到 WebView2。

MAS.Charts/HTML

ChartAsset.html 是统一的图表渲染页面，ChartAsset.js 接收宿主消息，创建仪表盘网格、摘要卡片和 ECharts 实例，并处理主题、加载状态、错误状态及窗口大小变化。

资产定义

ChartAsset 的根模型是 ChartAssetDefinition：

{
  "id": "production-overview",
  "name": "生产概览",
  "description": "按日期查看产量趋势",
  "parameters": [],
  "dataSources": [],
  "dashboard": {
    "columns": 2,
    "widgets": []
  },
  "refresh": {
    "mode": "manual",
    "intervalSeconds": 0
  }
}

根字段：

字段 说明
id 项目内稳定且唯一的资产标识
name 资产显示名称
description 可选说明
parameters 参数定义集合
dataSources 数据源定义集合
dashboard 仪表盘布局与组件集合
refresh 手动或定时刷新策略

资产保存在当前 ChartAsset 项目的 ChartAssetProjectModel.Assets 中，不是单独的 SQL 或 HTML 文件目录。

参数定义

参数模型为 ChartParameter：

{
  "name": "StartTime",
  "label": "开始时间",
  "type": "dateTime",
  "required": true,
  "defaultValue": "2026-07-01 00:00:00",
  "hint": "请选择查询开始时间",
  "options": null
}

字段说明：

字段 说明
name 参数名称，对应 SQL 中的 {{Name}}
label 页面输入控件的显示标签
type text、number、date、dateTime 或 enum
required 是否必填，默认 true
defaultValue 页面未提供值时使用的默认值
hint 输入提示文本
options enum 参数的候选值集合

参数名称只允许字母、数字和下划线，在同一资产内不区分大小写且不能重复。枚举参数必须提供候选项；数字、日期、日期时间和枚举的默认值会在资产校验时检查。

参数合并与类型转换

执行资产时，ChartParameterValidator.MergeAndValidate 根据参数定义合并调用方参数与默认值：

  1. 调用方提供的值优先；
  2. 未提供时使用 defaultValue；
  3. 必填参数没有有效值时产生校验错误；
  4. 枚举值必须属于 options；
  5. 校验通过后生成最终参数字典。

构建 SQL 命令时，ChartAssetSqlBuilder 根据参数类型调用 QueryAssetParsing.ConvertValue：

参数类型 绑定值
text 去除首尾空白后的字符串
enum 按文本绑定
number 优先转换为 long，其次转换为 decimal
date 解析为 DateTime.Date
dateTime 解析为 DateTime

如果数字或日期文本不能转换，转换函数会保留原始字符串；资产参数校验负责在执行前拦截无效输入。

数据源

数据源模型为 ChartDataSourceDefinition。每个数据源具有项目内资产范围唯一的 id，并使用 sql 或 script 模式。

通用字段：

字段 说明
id 数据源标识，供 widget 的 dataSourceId 引用
mode sql 或 script
sql SQL 模式的只读 SQL 模板
script Script 模式的 C# 源码
rowLimit 行数上限，小于等于 0 时默认 5000
timeoutSeconds 超时秒数，小于等于 0 时默认 30 秒

sql、script 和 optionJs 在 JSON 中既可以写成一个字符串，也可以写成字符串数组；字符串数组反序列化时以换行连接。

SQL 数据源

{
  "id": "trendData",
  "mode": "sql",
  "sql": [
    "SELECT production_date, quantity",
    "FROM production_record",
    "WHERE production_date >= {{StartDate}}",
    "  AND production_date < {{EndDate}}",
    "ORDER BY production_date"
  ],
  "rowLimit": 5000,
  "timeoutSeconds": 30
}

SQL 中的 {{参数名}} 必须在资产的 parameters 中声明。未声明的占位符会导致资产校验失败。

执行时占位符被转换为命名参数：

WHERE production_date >= @StartDate
  AND production_date < @EndDate

实际值通过数据库命令参数绑定，不直接拼入 SQL 文本。

Script 数据源

Script 模式用于多次查询、聚合或需要 C# 处理的数据源。script 源码必须实现 IChartAssetScript。

运行时向脚本提供 ChartScriptContext，其中包含：

  • 已合并的资产参数；
  • 当前数据源行数限制；
  • 可供脚本调用的查询函数。

脚本查询同样通过 PreparedSqlCommand 和 IQueryExecutor 执行。脚本整体受数据源 timeoutSeconds 和调用取消令牌控制。

SQL 只读校验

SQL 数据源必须是一条只读语句。允许的首关键字为：

  • SELECT
  • WITH
  • SHOW
  • DESCRIBE
  • DESC
  • EXPLAIN

SQL 去除末尾分号后不能再包含分号。对于 SELECT 和 WITH，守卫会拦截 INSERT、UPDATE、DELETE、DROP、ALTER、CREATE、TRUNCATE、GRANT、REVOKE、REPLACE、MERGE、CALL、EXECUTE、EXEC、INTO、LOAD、HANDLER、COPY 和 VACUUM 等关键字。

SQL 守卫是执行前的文本校验。实际查询仍使用当前 ChartAsset 项目配置的数据库实例。

仪表盘

ChartDashboardDefinition 包含：

  • columns：网格列数，范围为 1 至 4；
  • widgets：按声明顺序排列的组件集合，至少需要一个组件。

每个 ChartWidgetDefinition 包含：

字段 说明
id widget 唯一标识
type chart 或 summaryCard
title 可选标题
dataSourceId 引用的数据源 ID
columnSpan 横向占用列数，不能超过仪表盘列数
height 像素高度；小于等于 0 时图表默认 320，摘要卡片自适应
option ECharts JSON option
optionJs 生成 ECharts option 的 JavaScript 源码
card 摘要卡片配置

图表 widget 的 option 与 optionJs 必须二选一。图表可以不引用数据源，此时作为静态图表渲染；摘要卡片必须引用数据源。

ECharts option 模式

option JSON

option 保存普通 ECharts option JSON。数据源执行成功后，后端将查询结果转换为 ECharts dataset 格式，并注入 option.dataset.source。

数据源结果：

{
  "columns": ["production_date", "quantity"],
  "rows": [
    ["2026-07-01", 120],
    ["2026-07-02", 135]
  ]
}

注入后的 dataset source：

[
  ["production_date", "quantity"],
  ["2026-07-01", 120],
  ["2026-07-02", 135]
]

option 中可使用 encode 和 seriesLayoutBy 按列引用数据。

option JSON 内需要 ECharts 回调函数时，字符串使用 @js: 前缀：

{
  "tooltip": {
    "formatter": "@js:function (params) { return params[0].name; }"
  }
}

前端会递归查找 @js: 字符串，通过 new Function 编译为函数。dataset 子树不会参与函数恢复，防止查询数据被当作代码执行。

optionJs

optionJs 是一段由统一前端执行的 JavaScript 语句，必须给预声明的 option 变量赋值：

{
  "optionJs": [
    "const labels = data.rows.map(row => row[0]);",
    "const values = data.rows.map(row => row[1]);",
    "option = {",
    "  xAxis: { type: 'category', data: labels },",
    "  yAxis: { type: 'value' },",
    "  series: [{ type: 'line', data: values }]",
    "};"
  ]
}

前端通过 new Function('data', ...) 执行源码。data 结构为：

{
  columns: ["production_date", "quantity"],
  rows: [["2026-07-01", 120]],
  source: [["production_date", "quantity"], ["2026-07-01", 120]]
}

没有数据源时 data 为 null。如果脚本没有设置 option.dataset 且存在数据，前端自动注入 dataset.source。

后端只检查 optionJs 是否满足资产结构要求，不执行 JavaScript；语法错误和运行错误在 WebView2 渲染时显示为 widget 错误。

摘要卡片

summaryCard 从数据源结果的第一行提取显示值，不创建 ECharts 实例。

ChartSummaryCardDefinition 的主要字段：

字段 说明
valueColumn 主数值列名；为空时使用第一列
unit 数值后的单位
precision 小数位数，范围 0 至 6；为空时保留原值
trendText 静态趋势文字
trendTextColumn 从结果列读取趋势文字
trendDirection 静态方向：flat、up、down
trendDirectionColumn 从结果列读取方向
tone default、success、warning、danger 或 info

趋势文字的静态值与结果列配置二选一；趋势方向也采用相同规则。方向列可以返回 up、down、flat，也可以返回数值并按正负号判断方向。

运行时执行流程

ChartAssetRuntime.ExecuteAsync 的执行顺序如下：

  1. 根据资产 ID 从当前项目查找资产；
  2. 调用 ChartAssetValidator 重新校验资产；
  3. 合并调用参数与参数默认值并校验；
  4. 当资产包含数据源时，读取项目数据库配置并设置查询执行器；
  5. 按资产声明顺序逐个执行数据源；
  6. SQL 数据源构建参数化命令并查询数据库；
  7. Script 数据源编译并在超时限制内执行；
  8. 每个数据源结果保存为 ChartDataResult，异常保存为该数据源的错误字符串；
  9. 遍历 dashboard widgets，将数据源结果转换为 option、optionJs 数据或摘要卡片；
  10. 组装并返回 ChartRenderPayload。

某个数据源执行失败时，运行时不会终止其他数据源。引用该数据源的 widget 在 Payload 中带有 error，其他 widget 仍可正常显示。调用取消会终止整个执行过程。

查询结果转换

数据库查询结果从 DataTable 转换为 ChartDataResult：

{
  "columns": ["name", "quantity"],
  "rows": [["A", 10], ["B", 20]],
  "rowCount": 2,
  "elapsedMilliseconds": 12
}

单元格转换规则：

  • null 和 DBNull 转换为 null；
  • DateTime 格式化为 yyyy-MM-dd HH:mm:ss；
  • TimeSpan 转换为字符串；
  • byte[] 转换为 null；
  • 其他类型保留原值。

渲染 Payload

运行时返回 ChartRenderPayload：

{
  "assetId": "production-overview",
  "name": "生产概览",
  "description": "按日期查看产量趋势",
  "columns": 2,
  "widgets": [
    {
      "widgetId": "trend",
      "type": "chart",
      "title": "产量趋势",
      "columnSpan": 2,
      "height": 320,
      "option": {},
      "optionJs": null,
      "datasetSource": null,
      "card": null,
      "rowCount": 31,
      "elapsedMilliseconds": 16,
      "error": null
    }
  ],
  "generatedAt": "2026-07-11T10:00:00"
}

ChartAssetWebViewService 将 Payload 包装为 WebView2 消息：

{
  "type": "chart-asset-render",
  "payload": {}
}

其他宿主消息包括：

type 作用
chart-asset-theme 切换页面明暗主题
chart-asset-chart-theme 切换 ECharts 主题
chart-asset-loading 显示加载状态
chart-asset-error 显示页面级错误
chart-asset-reset 清空当前渲染内容

前端收到新 Payload 时释放旧的 ECharts 实例，重新创建 widget DOM，并为 chart widget 调用 echarts.init 和 setOption。

主题与布局处理

ChartAsset 前端使用 CSS 网格根据 columns 和 columnSpan 排列组件。对于直角坐标系图表，如果 option 没有显式设置 grid，前端自动添加带 containLabel 的默认网格，避免坐标文字被裁剪。

前端从当前页面 CSS 变量读取文字、弱化文字、边框和背景色，并为 option 中未显式配置的标题、图例、坐标轴、提示框等补充主题默认值。资产显式设置的 option 值优先，不会被主题默认值覆盖。

刷新策略

刷新模型为：

{
  "mode": "interval",
  "intervalSeconds": 10
}
  • manual：只在用户选择资产、修改参数或主动刷新时执行；
  • interval：首次执行成功后由 ChartAssetViewModel 创建定时器重复执行；
  • 定时刷新间隔最小为 5 秒；
  • 切换资产、离开页面或释放 ViewModel 时停止原定时器。

资产校验

ChartAssetValidator 在保存和执行前检查：

  • 资产 ID 和名称；
  • ID、名称及参数重复；
  • 参数名称、类型、默认值与枚举选项；
  • 数据源 ID、模式和内容；
  • SQL 只读性及占位符声明；
  • Script 是否声明实现 IChartAssetScript；
  • dashboard 列数和 widget 数量；
  • widget ID、跨度和数据源引用；
  • chart 的 option / optionJs 二选一；
  • summaryCard 的字段组合；
  • @js: 函数字符串格式；
  • 刷新模式和最小间隔。

校验结果包含 errors 和 warnings。存在错误时资产不能保存或执行；警告不会阻止保存。

IPC 工具

chartAsset.list

列出当前项目资产，支持 search、offset 和 limit。返回资产 ID、名称、说明、参数数量、数据源数量、widget 数量和刷新模式。

chartAsset.get

按 ID 或名称返回完整资产 JSON。maxJsonChars 控制返回文本截断长度。

chartAsset.validate

校验未保存或已存在资产。输入可使用：

  • draftId：草稿存储中的资产 JSON；
  • assetJson：直接传入完整定义；
  • assetIdOrName：校验项目中已有资产。

三种来源只能选择一种。

chartAsset.upsert

使用 draftId 或 assetJson 创建或替换资产。资产按 ID 匹配；allowReplace 默认为 true。写入前执行完整校验，成功后触发资产变更事件。

chartAsset.patch

对已有资产执行局部 JSON 修改。支持：

  • replace
  • remove
  • append
  • insertAfter

路径使用 JSON Pointer，并支持数组的 byId/{id} 定位。补丁不能修改资产 ID；合并后的完整资产通过校验后才会保存。

chartAsset.delete

按资产 ID 或名称从项目中删除资产。ChartAsset 删除不是 ViewAsset 的软注销，不提供 restore 工具。

chartAsset.preview

执行资产并返回 widget 渲染摘要：

{
  "assetIdOrName": "production-overview",
  "parameters": [
    { "name": "StartDate", "value": "2026-07-01" },
    { "name": "EndDate", "value": "2026-08-01" }
  ],
  "includePayload": true,
  "maxJsonChars": 60000
}
  • parameters 由名称和值组成，工具层转换为不区分大小写的字典；
  • includePayload=false 时主要返回每个 widget 的行数、耗时和错误摘要；
  • includePayload=true 时同时返回完整 ChartRenderPayload；
  • maxJsonChars 控制完整 Payload JSON 的最大返回长度。

当前能力边界

  • 一个资产支持多个参数、多个数据源和多个 widget；
  • SQL 数据源按声明顺序逐个执行，不并行执行；
  • 支持只读 SQL 与受控 C# Script 两种数据源；
  • SQL 参数必须先在资产参数集合中声明；
  • 支持 text、number、date、dateTime 和 enum 参数；
  • 支持 ECharts option JSON、@js: 回调函数和完整 optionJs；
  • 支持 chart 与 summaryCard 两种 widget；
  • 支持 1 至 4 列仪表盘和 widget 跨列；
  • 支持手动刷新及最小 5 秒的定时刷新；
  • 单个数据源失败只影响引用它的 widget；
  • ChartAsset 使用统一 WebView2 页面渲染，不提供每个资产独立的 HTML/CSS 页面；
  • ChartAsset 工具用于管理项目内的 JSON 定义，不操作 ViewAsset 文件目录。
