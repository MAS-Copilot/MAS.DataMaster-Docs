图形渲染

概述

图形渲染模块负责将解析后的几何数据或三维模型可视化显示在视口中，并提供交互式浏览能力

该模块基于三维视口组件构建，负责场景组织、相机控制、用户交互与辅助信息绘制

其目标是在保证渲染性能的前提下，提供稳定、流畅的三维查看体验，用于路径预览、模型检查与调试

图形渲染流程主要包含以下阶段：

  1. 场景初始化
  2. 模型加载
  3. 相机控制
  4. 实时渲染更新
  5. 辅助信息绘制

场景结构

三维场景采用统一的节点结构进行管理

SceneRoot：

场景根节点，用于承载所有三维对象。

ImportedModel：

模型节点，用于存放加载的三维模型或解析后的几何对象。

Scene Behavior：

场景行为节点，用于添加辅助元素，例如背景效果或参考对象。

通过这种结构可以实现：

  • 模型数据与辅助元素分离
  • 支持模型替换与追加加载
  • 便于扩展新的渲染元素

模型加载

模型加载负责将外部数据转换为可渲染对象，并加入场景

支持两种输入形式：

  • 模型文件
  • 已解析的模型对象

加载方式支持两种模式：

替换模式：

清空现有模型后加载新模型

适用于：

  • 单模型查看
  • 切换图纸或工件

追加模式：

保留已有模型并追加新模型

适用于：

  • 多对象对比
  • 叠加显示路径或辅助元素

相机控制

视口使用透视相机进行场景观察

相机维护以下核心参数：

  • Position：相机位置
  • LookDirection：观察方向
  • UpDirection：上方向
  • TargetPoint：观察目标点
  • Distance：相机与目标点距离
  • FieldOfView：视场角

相机控制提供以下能力：

  • 视角旋转
  • 视角平移
  • 视距缩放
  • 相机重置
  • 飞行到指定视角
  • 自适应视图

第一人称模式（FirstPerson）

第一人称模式用于在场景中自由移动观察

操作方式：

  • 鼠标移动：控制视角旋转
  • W / A / S / D：前后左右移动
  • Space / C：上下移动
  • Shift：移动加速
  • 鼠标左键：临时缩小视场角进行观察

该模式适用于：

  • 场景漫游
  • 设备内部结构查看
  • 大型模型浏览

第三人称模式（Orbit）

第三人称模式用于围绕目标点观察模型

操作方式：

  • 右键拖动：轨道旋转
  • 中键拖动：平移视图
  • 滚轮：缩放距离
  • 左键：临时缩小视场角

该模式适用于：

  • 模型检查
  • 路径预览
  • 工件姿态观察

缩放（Zoom）

缩放通过改变相机与目标点之间的距离实现

缩放过程采用指数缩放方式，使缩放过程更加平滑

同时引入缩放惯性：

  • 鼠标滚轮产生缩放冲量
  • 每帧逐渐衰减
  • 形成平滑缩放效果

轨道旋转（Orbit）

轨道旋转用于围绕目标点观察模型

旋转过程通过以下步骤实现：

  1. 计算相机到目标点的向量
  2. 根据鼠标位移计算偏航角与俯仰角
  3. 更新相机位置
  4. 更新观察方向与上方向

旋转结束后保留旋转速度，并逐帧衰减，形成惯性旋转效果

平移（Pan）

平移用于在保持视角方向不变的情况下移动观察区域

平移过程：

  1. 根据当前相机方向计算右方向与上方向
  2. 将鼠标位移转换为世界坐标位移
  3. 同时移动相机位置与目标点

相机重置（Camera Reset）

相机重置用于恢复初始视角

重置过程：

  1. 记录当前相机状态
  2. 使用缓动函数进行插值
  3. 在指定时间内逐帧更新相机参数
  4. 恢复初始相机位置、方向和距离

该过程使用平滑动画，避免视角突变

视角飞行（View Animation）

视角飞行用于平滑移动到指定视角

执行流程：

  1. 记录当前相机状态
  2. 设定目标相机状态
  3. 在指定时间内进行插值
  4. 每帧更新相机位置与方向

该功能常用于：

  • 视角预设切换
  • 自动定位模型
  • 视图导航

实时渲染

渲染系统使用逐帧更新机制。

每帧执行以下更新：

  1. 更新背景或场景行为
  2. 更新相机移动
  3. 更新轨道旋转惯性
  4. 更新缩放惯性
  5. 更新相机动画
  6. 更新辅助显示

渲染系统支持：

  • 启动实时渲染
  • 停止实时渲染

用于控制后台资源消耗

坐标轴（Axes）

系统提供两种坐标轴显示：

世界坐标轴：

位于三维场景中，用于表示全局坐标方向。

参考坐标轴：

绘制在屏幕角落，用于指示当前视角方向

辅助信息显示

视口支持绘制调试信息

显示内容包括：

  • FPS
  • 相机位置
  • 目标点坐标
  • 相机距离
  • 偏航角与俯仰角
  • 当前视场角

该信息主要用于调试与开发阶段

设计模式（Design Mode）

在设计模式下不会执行真实渲染逻辑，而是显示占位信息

其作用包括：

  • 避免设计器加载三维资源
  • 提供组件占位显示
  • 提高设计器稳定性
