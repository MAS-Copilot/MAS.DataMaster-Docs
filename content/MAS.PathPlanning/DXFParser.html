几何实体解析

文本注解解析

系统支持在 DXF 图层中通过文本注解定义原点与扩展点，用于辅助路径规划

注解解析对大小写不敏感，并按图层分别处理

注意事项

  • 文本实体分为 Text（单行文本）和 MText（多行文本）
  • 原点当前仅从 Text 实体中解析
  • 扩展点同时支持从 Text 与 MText 实体中解析
  • 当 MText 包含格式控制符时，系统会先去除富文本标记，再进行匹配
  • 注解仅作为辅助语义数据，不参与轨迹几何本体的构建

注解格式

标题 文本格式 描述 是否必须 支持实体类型
原点 Origin: X = value, Y = value, Z = value 设置该图层的原点坐标，用于相对坐标计算 否 Text
原点（2D） Origin: X = value, Y = value 设置该图层的二维原点，Z 默认为 0 否 Text
扩展点 Name-Index: X, Y, Z 设置该图层的扩展功能点坐标 否 Text / MText

原点

  • 原点坐标通过解析当前图层中的 Text 文本提取
  • 支持 2D 和 3D 两种格式
  • 若同一图层存在多个符合规则的原点文本，则按遍历顺序返回第一个匹配项
  • 若未定义原点，则使用默认值 (0, 0, 0)

示例：

  • 3D：Origin: X = 10.0, Y = 20.0, Z = 0.0
  • 2D：Origin: X = 10.0, Y = 20.0

扩展点

  • 扩展点坐标通过解析当前图层中的 Text 或 MText 文本提取
  • 扩展点名称支持中英文映射，解析后统一转换为系统内部标准名称
  • 同一图层内允许存在多个扩展点
  • 若同一图层中出现重复的 Name-Index 或重复 Index，仅保留首次解析成功的点
  • 扩展点解析结果包含：
    • 点名称 PointName
    • 点序号 Index
    • 绝对坐标 AbsolutePoint
    • 点类型 PointType
    • 数据来源 Source=Parse
  • 扩展点解析完成后，会基于图层原点进一步计算相对坐标

扩展点名称映射

  • 拍照点 => PhotoPoint
  • PhotoPoint => PhotoPoint
  • 抓取点 => GrabPoint
  • GrabPoint => GrabPoint
  • 放置点 => PlacementPoint
  • PlacementPoint => PlacementPoint
  • 结合点 => CombinePoint
  • CombinePoint => CombinePoint
  • 作业点 => OperationPoint
  • OperationPoint => OperationPoint

格式要求

扩展点文本格式如下：

Name-Index: X, Y, Z

其中：

  • Name 为扩展点名称，支持中英文
  • Index 为正整数编号
  • X, Y, Z 为三维坐标，按顺序分别表示 X、Y、Z

示例：

  • 拍照点-1: 10.0, 20.0, 0.0
  • PhotoPoint-2: 10.0, 20.0, 0.0
  • GrabPoint-3: -5.5, 12.0, 8.0

组

系统会读取 DXF 中的 Group 信息，并保存组名称与描述等元数据

当前行为

  • 当前阶段仅解析并保留组信息
  • 组信息已写入解析结果中，供后续业务使用
  • 当前这部分代码尚未基于组信息对几何实体执行过滤或忽略处理

后续用途

组（Group）可用于组织图层中的几何体，以实现更精细的解析控制。例如：

  • 标记某些几何体为辅助线
  • 标记某些几何体为忽略对象
  • 控制路径规划时参与计算的实体范围

CAD 使用建议

在 CAD 软件中，可将不希望参与后续路径规划的几何体划分到特定组中，供后续解析阶段识别和过滤

  • 选择需忽略的几何体
  • 右键创建组
  • 为后续解析规则保留该组信息
