Option

获取或设置 ECharts option JSON 配置；仅 Chart 类型使用， 与 OptionJs 二选一

public JObject? Option { get; init; }

Property Value

JObject

Remarks

以 JSON 配置为主；需要函数（如 formatter）时，用 FUNCTION_PREFIX（@js:）前缀的字符串携带 JS 函数源码， 由前端在渲染前编译（dataset 子树是纯数据，不参与编译）。 数据源结果由运行时注入到 option.dataset.source， 建议 option 使用 encode / seriesLayoutBy 引用数据列

OptionJs

获取或设置 ECharts option 的原生 JS 源码；仅 Chart 类型使用， 与 Option 二选一

[JsonConverter(typeof(StringOrLinesJsonConverter))]
public string? OptionJs { get; init; }

Property Value

string

Remarks

源码是一段 JS 语句（可含变量声明、辅助函数），必须给已预声明的 option 变量赋值。 后端不解析、不执行，原样推送给前端，由前端以 new Function('data', ...) 求值后 setOption。 数据源结果通过 data 形参（{columns, rows, source}，无数据源时为 null）提供； 脚本未设置 option.dataset 时，前端自动注入 dataset.source。 JSON 中可写成字符串，也可写成按行拆分的字符串数组（反序列化时以换行符连接）， 后者更适合在 JSON 里携带多行源码
