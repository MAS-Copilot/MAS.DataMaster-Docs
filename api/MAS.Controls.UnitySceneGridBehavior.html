Class UnitySceneGridBehavior

Namespace
MAS.Controls
Assembly
MAS.Controls.dll

Unity Scene View 风格的自适应网格背景（线几何 + 矩阵变换实现）。

网格由三组互不重叠的 HelixToolkit.Wpf.SharpDX.LineGeometryModel3D 组成： 按线的整数索引 k 分类——k 为 100 倍数进次主级组、10 倍数进主级组、 其余进细级组。三组之间没有任何重合线段，也不存在叠加的透明平面， 从根源上消除 Z-fighting、阴影与闪烁。

顶点在"单位索引空间"（间距 = 1）构建，缩放与平移通过组节点的 缩放×平移 矩阵完成；窗口中心吸附到 100 格的整数倍，保证平移后 各线的模分类不变。常规视角下每帧最多更新一个 4×4 矩阵与三个线色。

近平面裁剪：HelixToolkit 的线几何着色器把端点做透视除法后在屏幕空间 展开线宽，不处理端点落在相机平面附近（w≈0）的情况——除以趋零的 w 产生无穷大/NaN，整条线随机丢帧（低端显卡上尤其明显）。因此当网格窗口 伸到相机近平面之后时，在 CPU 侧按近平面对每条线做一维区间裁剪后重建， 保证任何端点都严格位于相机前方；窗口完全在相机前方时跳过裁剪、零重建。

缩放自适应：以相机视线与网格平面交点处的"每像素世界单位"推导目标间距， 取 log10 得到离散层级（细级间距 = MinSpacing × 10^level）；层级的小数部分 经 smoothstep 后作为混合因子——细级线淡出、主级线向细级亮度过渡、 次主级恒定，换级瞬间三组亮度首尾衔接、无跳变。"透明度"通过把线色向 背景色插值实现，渲染始终不透明——不依赖透明混合与绘制顺序。
